# Rip Highlight - @rip-lang/highlight

> **The highlight.js grammar for Rip — one shared definition consumed by Rip Print and the print editor extension.**

One file registers the Rip language with highlight.js. Every surface
that highlights Rip through hljs consumes this definition — the
`rip-print` CLI imports it directly, and the VS Code extension ships a
generated CJS copy that its tests byte-gate against this source — so
the grammar can never fork.

**Runtime:** browser-safe (`rip.browser: true`). One `.js` file with
zero imports; `highlight.js` itself is the consumer's dependency.

## Quick Start

```bash
bun add @rip-lang/highlight
```

```coffee
import hljs from 'highlight.js/lib/core'
import rip from '@rip-lang/highlight'

hljs.registerLanguage 'rip', rip
html = hljs.highlight('tags = %w[alpha beta]', { language: 'rip' }).value
```

## Features

- **Keywords and declarations** — control flow, `def` (with function
  names), `class`, `schema`, `component`/`render`/`slot`/`offer`/`accept`
- **Strings** — double/single quotes and heredocs, with `#{}`
  interpolation where Rip interpolates
- **Word arrays** (`%w[…]` and paired-delimiter variants) and
  **symbol literals** (`:name`)
- **Rip operators** — reactive (`:=`, `~=`, `~>`), two-way bind
  (`<=>`), method/merge/readonly assignment (`.=`, `*>`, `=!`),
  prototype `::`, dammit `!`, tagged-template `$`
- **Regexes, heregexes, numbers, comments** — including hex numbers
  and block comments

## Consumers

| Surface | How it consumes |
| --- | --- |
| `@rip-lang/print` (CLI) | imports this module directly |
| `packages/print/vscode` | ships `lib/hljs-rip.js`, generated by its `rip sync.rip` and byte-gated by its tests |

A grammar change lands here once; the extension regenerates with
`bun run sync` from `packages/print/vscode` (its suite fails until it
does).

## Test

```bash
bun run test
```

The suite pins the export surface and browser safety, then drives the
grammar through highlight.js: keywords, strings and interpolation
rules, word arrays, symbols, regexes, numbers, comments, and the Rip
operator set.
